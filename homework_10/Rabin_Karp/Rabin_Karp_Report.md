## Rolling hash

Rolling hash позволяет быстро пересчитывать хеш подстроки при сдвиге окна без полного пересчета:
- сначала вычисляется хеш образца и первого окна текста;
- при сдвиге окна удаляется вклад выбывающего символа, хеш умножается на base, добавляется вклад нового символа;
- все операции берутся по модулю простого числа, чтобы значения оставались в допустимом диапазоне;
- при совпадении хешей выполняется прямое сравнение строк, чтобы устранить влияние коллизий.

Формула для перехода от предыдущего окна к следующему:
```
next_hash = (base * (old_hash - ord(text[i]) * power) + ord(text[j+1])) % mod
```
Если результат отрицательный, к нему добавляется mod.


## Оценка сложности

- Средний случай: O(n + m). Начальные хеши считаются за O(m), каждое последующее окно обновляется за O(1).
- Худший случай: O(n * m), когда из-за коллизий приходится сравнивать подстроки посимвольно.

Здесь n — длина текста, m — длина паттерна. Вероятность коллизий определяет, насколько часто поведение приближается к худшей оценке.
