## Алгоритм Кнута–Морриса–Пратта (KMP)

### Описание
KMP ищет подстроку за счет предварительной обработки шаблона. Для каждой позиции шаблона строится массив lps (longest proper prefix-suffix) — длина наибольшего собственного префикса, совпадающего с суффиксом для подстроки, заканчивающейся в этой позиции. Это позволяет при несовпадении символов не возвращаться в начало шаблона, а перепрыгивать на позицию, заданную lps.

### Построение массива lps
1. Инициализация: lps[0] = 0, указатель length = 0.
2. Для каждой позиции i:
   - если pattern[i] == pattern[length], увеличиваем length, записываем ее в lps[i], двигаемся дальше;
   - иначе, если length > 0, откатываемся к lps[length-1] и продолжаем сравнение;
   - если length == 0, записываем 0 и переходим к следующей позиции.

### Поиск в тексте
1. Идем по тексту индексом i и по шаблону индексом j.
2. При совпадении символов двигаем оба индекса. Если j дошел до конца шаблона — найдено вхождение.
3. При несовпадении:
   - если j > 0, откатываем j на lps[j-1] и продолжаем без сдвига i;
   - если j == 0, просто двигаем i.

### Сложность
- Построение lps: O(m), где m — длина шаблона.
- Поиск: O(n), где n — длина текста.
- Итоговая сложность: O(n + m), дополнительная память O(m) на lps.
